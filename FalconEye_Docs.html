Core Concept

FalconEYE is an AI-powered security code analysis tool that uses local Large Language Models (via Ollama) to identify security vulnerabilities. Unlike traditional security scanners that rely on pattern matching, FalconEYE uses pure AI reasoning to understand code context and identify both known and novel vulnerabilities.

The tool implements Retrieval-Augmented Generation (RAG) to provide rich context to the AI, including similar code segments, documentation, and structural metadata (AST). This enables highly accurate, context-aware security assessments with reduced false positives.

Key Differentiators

Pure AI Reasoning

No pattern matching - AI understands code semantics and identifies vulnerabilities through reasoning

Privacy-First

All analysis runs locally using Ollama - your code never leaves your machine

RAG-Powered

Retrieves similar code and documentation to provide rich context for accurate analysis

Multi-Language

Supports 10+ languages with extensible plugin system for language-specific security knowledge

Smart Re-indexing

Tracks file changes and only re-analyzes modified code for faster subsequent scans

High Accuracy

Optional AI validation step filters false positives and confirms real vulnerabilities

Architecture › Hexagonal Architecture

Hexagonal Architecture (Ports & Adapters)

FalconEYE follows the Hexagonal Architecture pattern, which separates the application into distinct layers: the domain core (pure business logic), application layer (use case orchestration), adapters (external interfaces), and infrastructure (external system integrations). This architecture ensures clean separation of concerns, high testability, and flexibility to swap implementations without affecting core business logic.

Adapters Layer - User interfaces & I/O
Application Layer - Use case orchestration
Domain Layer - Pure business logic
Infrastructure Layer - External integrations

Key Component Relationships

CLI → Application Handler

The CLI commands in adapters/cli/commands.py call application handlers (IndexCodebaseHandler, ReviewFileHandler) to execute use cases. CLI is responsible only for parsing arguments and displaying results.

Application Handler → Domain Services

Application handlers orchestrate multiple domain services to complete a workflow. For example, ReviewFileHandler coordinates LanguageDetector, ContextAssembler, and SecurityAnalyzer to perform a complete security review.

SecurityAnalyzer → LLMService → OllamaAdapter

SecurityAnalyzer (domain) calls LLMService (domain interface), which is implemented by OllamaLLMAdapter (infrastructure). This allows swapping LLM providers without changing business logic.

ContextAssembler → VectorStore + MetadataRepository

ContextAssembler performs semantic search via VectorStoreRepository and retrieves AST metadata via MetadataRepository to build rich context for AI analysis.

Architecture › Core Workflows

Core Workflows

1. Index Codebase Workflow

Command: falconeye index /path/to/project

Project Identification
ProjectIdentifier generates stable project ID from git remote or directory path
Supports monorepo isolation with explicit project IDs
Language Detection & File Discovery
LanguageDetector scans for all supported languages in codebase
Multi-language file discovery (respects .gitignore)
Smart Re-indexing
ChecksumService compares current files with cached metadata
Identifies new, changed, and deleted files
Only re-analyzes changed files for performance
AST Analysis & Metadata Extraction
EnhancedASTAnalyzer uses tree-sitter for multi-language parsing
Extracts functions, classes, imports, variables
Stores structural metadata for RAG context
Vector Storage
ChromaVectorStore stores embeddings in project-scoped collection
Separate collections for code and documentation

2. Security Review Workflow

Command: falconeye review /path/to/file.py

File Reading & Language Detection
Read target file content
LanguageDetector identifies programming language
RAG Context Assembly
Semantic search in vector store for similar code (top_k=5)
Retrieve AST metadata (functions, imports, data flow)
AI Security Analysis
OllamaLLMAdapter sends to qwen3-coder:30b
AI reasons about security vulnerabilities
Result Formatting & Output
Output to console, JSON, HTML, or SARIF
Save report if output_directory configured

Architecture › Key Features

Key Features

AI-Powered Analysis

Pure AI reasoning using large language models. No pattern matching - understands code semantics and identifies both known and novel vulnerabilities.

RAG-Powered Context

Semantic search retrieves similar code and documentation to provide rich context for AI analysis, enabling multi-file understanding.

Smart Re-indexing

Tracks file changes using checksums. Only re-analyzes changed or new files, dramatically reducing scan time.

Multi-Language Support

Supports 10+ languages with extensible plugin system. Each language has specialized prompts and security knowledge.

Privacy-First

All analysis runs 100% locally using Ollama. Your code never leaves your machine. No external API calls.

High Accuracy

Optional AI validation step re-evaluates findings to filter false positives and confirm real vulnerabilities.

Rich Output Formats

Console, JSON, HTML, and SARIF formats for different use cases - from interactive terminals to CI/CD integration.

Production-Ready

Circuit breaker, retry logic, structured logging, and graceful error handling for production environments.
